Rule Authoring Guide

Everything you need to write custom tree-sitter rules and wire them to your organisation.

1. Rule file format

Rules are stored in a YAML file with a top-level rules list. Each entry is one rule. The file can live anywhere in your rules repository — by convention it is named rules.yaml at the repo root. 

rules:
  - id: js-no-eval
    name: No eval()
    severity: error
    language: javascript
    query: |
      (call_expression
        function: (identifier) @fn
        (#eq? @fn "eval"))
    message: "eval() executes arbitrary strings as code and is a critical security risk"

  - id: js-no-alert
    name: No alert/confirm/prompt
    severity: warning
    language: javascript
    query: |
      (call_expression
        function: (identifier) @fn
        (#match? @fn "^(alert|confirm|prompt)$"))
    message: "Browser dialogs block the thread and should not reach production"

2. Field reference

Field Required Description
id Yes Unique identifier. Appears in SARIF output and findings table. Convention: lang-short-description (e.g. js-no-eval).
name Yes Human-readable rule name shown in reports.
severity Yes One of error, warning, or note. Maps to the SARIF level field.
language Yes Target language (case-insensitive). See §4 for the full list.
query Yes Tree-sitter S-expression query. Every capture (@name) that matches produces one finding. See §5.
message Yes Explanation shown alongside each finding. Should explain the risk and suggest a fix.
metadata No Arbitrary JSON/YAML object. Use it for ticket links, CWE references, etc. Not interpreted by the scanner.

3. Severity levels

error

Blocks merge when fail_on_severity includes "error". Use for security vulnerabilities and critical correctness issues.

warning

Visible in the findings table. Blocks merge only if fail_on_severity includes "warning". Use for style and best-practice violations.

note

Informational only. Never blocks merge unless explicitly configured. Use for suggestions and deprecation notices.

Adjust which severities fail scans in Settings → Policy JSON: 

{
  "fail_on_severity": ["error", "warning"],
  "block_merge": true
}

5. Tree-sitter query primer

Queries are written in S-expression syntax. Each node type maps to a grammar node in the target language's tree-sitter grammar. Wrap a node with @capture_name to mark it as a finding location.

Match a function call by name

; Matches:  eval("code")
(call_expression
  function: (identifier) @fn
  (#eq? @fn "eval"))

Match a method call on a specific object

; Matches:  document.write(...)
(call_expression
  function: (member_expression
    object:   (identifier) @obj (#eq? @obj "document")
    property: (property_identifier) @prop (#eq? @prop "write")))

Match a property assignment

; Matches:  element.innerHTML = ...
(assignment_expression
  left: (member_expression
    property: (property_identifier) @prop
    (#eq? @prop "innerHTML")))

Match a comment containing a keyword

; Matches:  // @ts-ignore
(comment) @c
(#match? @c "@ts-ignore")

Match a JSX attribute (React)

; Matches:  <div dangerouslySetInnerHTML={...} />
(jsx_attribute
  (jsx_attribute_name) @attr
  (#eq? @attr "dangerouslySetInnerHTML"))

Python — function call

; Matches:  eval("code")   [Python grammar uses `call`, not `call_expression`]
(call
  function: (identifier) @fn
  (#eq? @fn "eval"))

6. Predicates

Predicates filter which captures produce findings. They must appear inside the same S-expression as the capture they reference.

Predicate Tests Example
#eq? Exact string equality (#eq? @fn "eval")
#match? Regex match (#match? @fn "^(alert|confirm)$")
Tip: Use #match? with anchors (^...$) to avoid partial matches. For example, #match? @fn "^eval$" will not match a function named evalSomething.

7. Example rules

js-no-settimeout-string error javascript

Catches setTimeout("code", ms) — passing a string is equivalent to eval(). 

- id: js-no-settimeout-string
  name: No setTimeout with string argument
  severity: error
  language: javascript
  query: |
    (call_expression
      function: (identifier) @fn
      (#match? @fn "^(setTimeout|setInterval)$")
      arguments: (arguments
        (string) .))
  message: "Passing a string to setTimeout/setInterval is equivalent to eval()"
ts-no-as-any error typescript

Catches value as any type assertions. 

- id: ts-no-as-any
  name: No 'as any' type assertion
  severity: error
  language: typescript
  query: |
    (as_expression
      type: (predefined_type) @type
      (#eq? @type "any"))
  message: "'as any' disables type checking entirely. Use a specific type assertion"
py-no-pickle-loads error python

Catches pickle.loads() on untrusted data. 

- id: py-no-pickle-loads
  name: No pickle.loads()
  severity: error
  language: python
  query: |
    (call
      function: (attribute
        object:    (identifier) @obj    (#eq? @obj "pickle")
        attribute: (identifier) @method (#eq? @method "loads")))
  message: "pickle.loads() on untrusted data enables arbitrary code execution"
team-no-console-log warning javascript · typescript

Ban console.log in production code. Add the rule twice — once per language — or create a shared rule file for each. 

- id: team-no-console-log
  name: No console.log
  severity: warning
  language: javascript
  query: |
    (call_expression
      function: (member_expression
        object:   (identifier) @obj  (#eq? @obj "console")
        property: (property_identifier) @prop (#eq? @prop "log")))
  message: "Remove console.log before merging"
  metadata:
    docs: "https://wiki.example.com/coding-standards#logging"

8. Connecting your rules repo

  1. Create a GitHub repository and commit your rules.yaml to its root.
  2. Open Settings and set Rules repository to owner/repo and Rules ref to the branch or tag to track (e.g. main).
  3. Optionally paste a GitHub PAT with contents:read scope if the repository is private. The PAT is stored encrypted (AES-256-GCM) and never logged.
  4. Fetch rules from CI using your API key: 
    curl -H "Authorization: Bearer $SCANNER_API_KEY" \
     https://your-server/api/v1/rules \
     -o merged-rules.yaml

scanner scan ./src --recursive --config merged-rules.yaml
    The server caches the rules file with ETag so subsequent calls skip the GitHub download when nothing has changed.
  5. Create an API key for each CI environment at Settings → API Keys. The key is shown once — copy it to your CI secrets immediately.
